Mapeo Empleado → Contrato

Query SQL para mapeo inicial:
-- Obtener empleados actuales y su asignación SELECT he.name as empleado, he.employee_type, he.contract_id, hc.analytic_account_id as contrato_actual, he.department_id, hd.name as departamento, CASE WHEN hd.name LIKE '%OYMA%' THEN 'OYMA_R1' WHEN hd.name LIKE '%VALLE%' THEN 'VALLE' WHEN hd.name LIKE '%BOYACA%' THEN 'BOYACA' WHEN hd.name LIKE '%ADMIN%' THEN 'ADMIN_CENTRAL' ELSE 'REVISAR_MANUAL' END as contrato_sugerido FROM hr_employee he LEFT JOIN hr_contract hc ON he.contract_id = hc.id LEFT JOIN hr_department hd ON he.department_id = hd.id WHERE he.active = true ORDER BY hd.name, he.name;

Paso 2: Asignar Empleados a Contratos

Script Python para asignación masiva:
import odoo # Conexión a Odoo env = odoo.api.Environment.from_loader() # Mapping departamentos → contratos dept_to_contract = { 'Operación OYMA': 'OYMA_R1', 'Mantenimiento OYMA': 'OYMA_R1', 'Proyectos Valle': 'VALLE', 'Operación Valle': 'VALLE', 'Gas Boyacá': 'BOYACA', 'Bucaramanga Dist': 'BUCARAMANGA', 'Zona 2 O&M': 'OYM_Z2', 'Nuevos Negocios': 'NUEVOS_NEGOCIOS', 'Florida Operación': 'FLORIDABLANCA', 'Administración': 'ADMIN_CENTRAL', 'Contabilidad': 'ADMIN_CENTRAL', 'Recursos Humanos': 'ADMIN_CENTRAL', 'Gerencia': 'ADMIN_CENTRAL' } # Obtener cuenta analítica por código def get_analytic_account(code): return env['account.analytic.account'].search([('code', '=', code)], limit=1) # Asignar cada empleado employees = env['hr.employee'].search([('active', '=', True)]) for emp in employees: dept_name = emp.department_id.name if emp.department_id else 'Administración' contract_code = dept_to_contract.get(dept_name, 'ADMIN_CENTRAL') analytic_account = get_analytic_account(contract_code) if analytic_account: # Actualizar contrato del empleado if emp.contract_id: emp.contract_id.analytic_account_id = analytic_account.id print(f"✅ {emp.name} → {contract_code}") else: print(f"❌ {emp.name} → No se encontró contrato {contract_code}")

Paso 4: Automatizar Distribución Contable

Regla de distribución automática en payslip:
# Código Python en regla salarial Odoo def compute_analytic_distribution(payslip): """ Distribuir automáticamente el gasto de nómina al contrato del empleado """ employee = payslip.employee_id contract = employee.contract_id if contract and contract.analytic_account_id: # 100% del gasto va al contrato asignado analytic_distribution = { str(contract.analytic_account_id.id): 100.0 } # Si el empleado trabaja en múltiples contratos (caso especial) if employee.multi_contract_distribution: analytic_distribution = employee.multi_contract_distribution return analytic_distribution else: # Por defecto: Admin Central admin_account = env['account.analytic.account'].search([ ('code', '=', 'ADMIN_CENTRAL') ], limit=1) return {str(admin_account.id): 100.0} if admin_account else {} # Aplicar en cada línea de nómina result.analytic_distribution = compute_analytic_distribution(payslip)

⚠️ Casos Especiales

Empleados Multi-Contrato

Problema: Algunos empleados (ej: supervisores) trabajan en múltiples contratos.

Solución: Campo en empleado para distribución porcentual:
# Ejemplo: Supervisor que trabaja 60% OYMA, 40% Valle employee.multi_contract_distribution = { 'OYMA_R1': 60.0, 'VALLE': 40.0 } # El sistema automáticamente distribuye: # Salario $10M → $6M a OYMA_R1, $4M a VALLE

Cambios de Contrato Durante el Mes

Lógica: Si empleado cambia de contrato a mitad de mes, distribución proporcional por días.
# Ejemplo: Cambio día 15 de BOYACA a VALLE days_contract_1 = 15 # días en BOYACA days_contract_2 = 15 # días en VALLE total_days = 30 distribution = { 'BOYACA': (days_contract_1 / total_days) * 100, # 50% 'VALLE': (days_contract_2 / total_days) * 100 # 50% }
